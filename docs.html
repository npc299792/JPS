JPS - Javascript Particle System


This is a lightweight modular Particlesystem for Javascript, the focus during the development was on efficiency, flexibility and usability. The main file is only 3kb and has just around 100 lines of code (without empty & comment lines) but offers all possibilities u need for creating your animations, from simple effects over scientific simulations upto full games!


Basic Implementation:

window.onload = function() {
	can = document.getElementById('myCanvas')
	ctx = can.getContext('2d')

	system = new JPS.SYSTEM()					//create the particlesystem
	render()
}

function render() {
	ctx.clearRect(0, 0, can.width, can.height)	//clear the canvas (if u want)
	system.render(ctx)							//render the particlesystem

	requestAnimationFrame(render)
}

Methods

spawn(count, ctx, props)Spawns the given count of particles in the system. The ctx (Canvas 2D context) is only required if u use it in a init() method in one of your modifiers. props is an optional Object with properties that will be copied into the generated particles before the init() methods are called. Returns a Array with the created Particles, if only 1 Particle is created it returns the Particle itself.
clear(particles, modifiers)Deletes all particles and/or modifiers from the system.
delete(index, ctx)Deletes the particle with the given index. If u want to delete one or multiple particles by its instance use the removeParticle() method. The ctx is only required if u need it in a exit() method in one of your modifiers.
render(ctx, time)Renders the whole system on the given ctx. time defines the passed time since the last rendering in milliseconds (Default: 16.66 = 60fps). This method also can called without the ctx parameter, be sure the render() methods of your modifiers are prepared for it.
addModifier(...args)Adds the given modifiers to the system. Also accepts Arrays with modifiers.

If a modifier has a init method and the system already has particles, the init method will be called with all existing particles. If a modifier needs the ctx in the init method u need to deliver it manually via the arguments:
system.addModifier(ctx, mod1, mod2, ...) // sets the ctx for all following modifiers
removeModifier(...args)Removes the given modifiers from the system. Also accepts Arrays with modifiers.
removeParticle(...args)Removes the given particles from the system. U can pass the particle objects itself or their index. Also accepts Arrays with particles or their indexes.

If one of your modifiers needs the ctx in the exit method u need to deliver it manually via the arguments:
system.removeParticle(ctx, p1, p2, ...) // sets the ctx for all following particles


Properties

NameDefaultDescription
x0The X-Offset of system in pixels. Moves the whole system left or right.
y0The Y-Offset of system in pixels. Moves the whole system up or down.
s1The scale of system (1 = 100%).
time0The Time the system is already running (in milliseconds).
speed1The animationspeed of the system (1 = 100%).
maxundefinedSets a limit of particles in the system. If this is 0, undefined or false, the amount is unlimited.
rendertimeundefinedThe rendertime in milliseconds. This does not includes the prev and post modifiers.
skip0Counts how many particles skipped the render process in the last rendering.
particlesArrayContains all particles of the system.
modifiersObjectContains 5 Array's to store the modifiers.



Particles


Each particle is a simple Javascript Object that will be stored in the system.particles Array. To be sure every particle will be correctly initiated u should always create them with the system.spawn() method.

NameDefaultDescription
x0The X-Position of the particle.
y0The Y-Position of the particle.
s1The scale of the particle (1 = 100%).
r0The rotation of the particle (in radient, Math.PI = 180Â°).
sx1The X-Scale of the particle.
sy1The Y-Scale of the particle.
rx0The X-Rotation of the particle.
ry0The Y-Rotation of the particle.
age0The time the particle already exist (in Milliseconds).
lifeundefinedThe lifetime of the particle (in Milliseconds). Is this undefined or 0, the particle will not removed automatically.
typeundefinedDefines a type for the particle, can be a Number, String or a Object. A Particle only interacts with modifiers of the same type, if this is undefined / null / false / 0, the particle will interact with all modifiers whatever type it has.

Modifiers


A modifier can be every Javascript Object that contains atleast 1 of the following functions...

NameDescription
init(particle, ctx, system)Will be invoked when the particle is initiated. All particle properties have their default values at that point.
render(particle, ctx, time, system)Will be invoked when the particle is rendered. time defines the past time since the last rendering as a multiplier. This function can return 2 values, if it's returning 1 the particle will be deleted, if it returns 2 the render process for this particle will be skipped. In both cases system.skip will be incremented.

Important: When this function is called the canvas is already fully transformed to the particle properties! If u draw something at the coordinates 0,0 it will be drawn at the x and y coordinates of the particle.
exit(particle, ctx, system)Will be invoked when the particle is being deleted.
prev(ctx, system, time)Will be invoked before the rendering starts.
post(ctx, system, time)Will be invoked after the rendering is done.


A modifier also can have these 2 optional functions...

NameDescription
add(system)Will be invoked when the modifier is added to a system.
remove(system)Will be invoked when the modifier is removed to a system.


Modifiers can be added directly via the constructor of the system class...

system = new JPS.SYSTEM(mod1, mod2, ...)

or managed afterwards with the addModifier and removeModifier methods...

system.addModifier(mod1, mod2, ...)

system.removeModifier(mod1, mod2, ...)


Order


Modifiers will be executed in the same order as they added, this can be important depending what the modifier does. Usally the following order is a good way...

1. Animation
2. Interaction
3. Drawing



Types


Every Modifier can have a type property to define the type of particle to interact with, this can be a Number, String or a Object. U also can define a array with a list of types to group them. A modifier with the type undefined / null / false / 0 will interact with all particles.


Create a Modifier


Before start your own modifier u should consider what exactly u wanna do and what entry points do u need. To build a simple modifier that moves rectangles over the screen we need the init and render entry points... 


myMod = {
	init: function(p)
		p.vx = Math.random() * 100						// set a random velocity
		p.y = Math.random() * can.height				// set a random height
	},
	render: function(p,ctx,time) {
		p.x += p.vx * time								// move the particle
		ctx.fillStyle = 'red'							// set the fillstyle
		ctx.fillRect(0, 0, 20, 20)						// draw the rectangle
		if (p.x < 0 || p.x > can.width) p.vx = -p.vx	// check if the rectangle is outbound
	}
}


The initialization... 


system = new JPS.SYSTEM(myMod)	// create the system
system.spawn(50)				// spawn 50 particles

Randomization


A important part of every Particlesystem is the randomization, without it all particles would exactly do the same. To support it the JPS Object contains a helpful function:

JPS.TOOLS.RND(value, range, integer)



JPS.TOOLS.RND()				// returns 0
JPS.TOOLS.RND(5)			// returns 5
JPS.TOOLS.RND(5,5)			// returns a value between 0 and 10
JPS.TOOLS.RND(20,10,1)		// returns a integer between 10 and 30

// u can also pass an array
JPS.TOOLS.RND([50,5])		// returns a value between 45 and 55
JPS.TOOLS.RND([100,50,1])	// returns a integer between 50 and 150

// pass an object to create a random color
JPS.TOOLS.RND({r:[100,100], a:[0.6,0.3]})
JPS.TOOLS.RND({r:255, g:[128,128], b:[128,128], a:1})

// if u pass an array that first entry is not a Number, it will choose a random entry of it
JPS.TOOLS.RND(['red','green','blue'])	// returns red, green or blue
JPS.TOOLS.RND([[15,5],[45,5]])			// returns a value between 10-20 or 40-50

Magnet



system = new JPS.SYSTEM(
	new JPS.LIB.EMITTER({x:w/2, y:h/2, w:w, h:h, rate:0, spawn:500}),
	(magnet = new JPS.LIB.MAGNET()),
	new JPS.LIB.CIRCLE(),
	new JPS.LIB.INFO()
)

document.addEventListener('mousedown', e => magnet.charge *= -1)
document.addEventListener('mousemove', e => {magnet.x = e.clientX; magnet.y = e.clientY})

Spacerace



system = new JPS.SYSTEM(
	new JPS.LIB.EMITTER({type:1, x:[w*.75,200], y:[h/2,100], rate:0, spawn:100}),

	(magnet = new JPS.LIB.MAGNET({type:1, id:'v', x:w/2, y:h/2, radius:50000, charge:-20, pcharge:5, pow:2})),
	new JPS.LIB.ANIMATE({type:1, x:0, y:[-150,1]}),
	new JPS.LIB.ALIGN(),
			
	new JPS.LIB.DRAG({exclude:{x:0, y:0, w:300, h:100}}),
	new JPS.LIB.TRIANGLE({type:1, w:[15,5], h:[8,4]}),
	new JPS.LIB.SPHERE({type:2, radius:[20,10]}),

	new JPS.LIB.SLIDER({prop:'speed', y:30, start:-5, end:5, text:'Speed: %%value'}),
	new JPS.LIB.SLIDER({target:magnet, prop:'charge', y:60, start:-100, end:0, text:'Charge: %%value'}),
	new JPS.LIB.INFO()
)

system.spawn(1, ctx, {type:2, radius:50, x:w/2, y:h/2}) // add a single star
